# 跨前任聚合规则

本文件定义如何将多位前任的特征数据聚合为统一的理想对象画像。

## 加权融合原则

**权重计算：**
```
weight = chemistry_score × quality_ratio
```

权重越高，该前任的特征在最终画像中影响越大。

**示例：**
- 前任A：chemistry=8.5，quality_ratio=0.70 → weight=5.95
- 前任B：chemistry=6.0，quality_ratio=0.40 → weight=2.40
- 前任A 的特征占最终画像的约 71%

## 数值维度融合

各性格维度评分取加权平均值：
```
merged_score = Σ(score_i × weight_i) / Σ(weight_i)
```

## 文本特征融合

对于列表类特征（共情表达、关心细节、倾听信号等）：
- 合并所有前任的列表
- 按出现频率排序，取 Top N
- 权重高的前任的词汇优先排列

## 冲突特征处理

当某维度在不同前任间差异 > 3 分时：
1. 在 persona.md 中标注 `⚠️ 偏好未定`
2. 展示两种风格的对话样本
3. 询问用户偏好哪种风格

**示例冲突提示：**
```
⚠️ 幽默感偏好未定：
- 前任A（高频幽默，8.5分）：常用「哈哈哈你这个人」「笑死我了」
- 前任B（低频幽默，4.0分）：话语认真，不常开玩笑
请问你更偏好哪种风格？
```

## 增量 Merge 规则（新增前任时）

当用户通过 `/analyze-ex` 新增一位前任：
1. 重新计算所有前任的权重
2. 更新各维度加权平均值
3. 新特征**追加**到列表，不覆盖已有内容
4. 版本号 +1
5. 旧版本自动备份

## 来源标注

融合后的画像中，特征来源可追溯：
- persona.md 的「前任贡献图谱」表格展示各前任的权重和主要贡献
- 不在画像内容中暴露具体前任信息（保护隐私）
