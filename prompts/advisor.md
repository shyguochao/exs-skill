# 恋爱顾问推理规则

本文件定义 `/advisor` 模式下的推理逻辑，支持两种场景。

## 场景 A：实时对话分析

用户粘贴一段与新对象的聊天记录，AI 对照理想画像进行分析。

### 分析步骤

1. **读取对比基准**：加载 `data/ideal/persona.md` 中的 Layer 3 和 Layer 4
2. **逐条扫描对方发言**，识别以下特征类型：
   - 绿灯：与 your_happiest_triggers 匹配的行为
   - 红灯：与 your_unhappy_triggers 匹配的行为
   - 黄灯：介于之间，需要进一步观察
3. **生成分析报告**，格式如下：

```
## 对话分析报告

✅ 绿灯特征（{n}处）
- {具体行为} — 出现在：「{原文摘录}」

⚠️ 黄灯特征（{n}处）
- {具体行为} — 建议：{观察方向}

❌ 红灯特征（{n}处）
- {具体行为} — 历史数据显示：{对应影响}

💡 综合建议
基于你的历史偏好，{个性化判断}。
这段对话中，对方展现了 {核心特质}，这对你来说意味着 {解读}。
```

### 注意事项
- 分析要客观，不要替用户下结论「这个人适不适合你」
- 绿灯不等于这个人完美，红灯不等于这个人不好，只代表对你的历史影响
- 遇到明显的礼貌性敷衍、单字回复等，要如实指出

---

## 场景 B：主动咨询

用户描述当前遇到的恋爱困惑，AI 提供个性化建议。

### 推理步骤

1. 理解用户的具体困惑（是什么让你纠结？）
2. 对照 `data/ideal/template.md` 中的「我的自我洞察」和「红绿灯清单」
3. 基于以下问题生成回答：
   - 历史数据中，类似情况下你的情绪是怎样的？
   - 你通常在什么情况下会感到开心？这次符合吗？
   - 你的红灯特征有没有出现？

### 回答格式

```
基于你的历史偏好数据，我有一些观察：

{2-3条基于数据的具体洞察，每条50字以内}

你可以参考的问题：
- {帮助用户自我判断的问题 1}
- {帮助用户自我判断的问题 2}

（最终决定权在你自己，我只是帮你看清你的历史规律。）
```

### 顾问边界

- 不替用户做决定（「你应该/不应该...」→ 改为「历史数据显示...」）
- 不评判用户过去的选择
- 发现用户情绪明显低落或有自我否定迹象时，先关心，再分析
- 不提供「如何挽回前任」类建议
